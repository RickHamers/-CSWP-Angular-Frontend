About this application

This application was made for the course 'Clientside web frameworks' at Avans Hogeschool Breda.

CSWP Angular Front-end (Pseudo-Tweakers.net/Reddit)

This Front-end serves as a knock-off forum, akin to tweakers.net & Reddit. You can create threads, on which people can comment. You can only edit or delete comments or threads if they are your own. However, you can comments on other people's threads as well as reply to other people's comments.You also have the ability to like or dislike (or upvote/downvote) threads and comments. You have a user profile which you can edit. You can change your password, for example.

Entities

User

  • Username: string (Unique)
  • Password: string

Thread

  • Title: string
  • Content: string
  • Author (Username): string
  • Upvote: number
  • Downvote: number
  • Comments: list of comment entities

Comment

  • Content: string
  • Thread-ID: number
  • Author (Username): string
  • Upvote: number
  • Downvote: number
  • Sub-comments: List of comment entities

Use Cases

The following functionality is available in this application.

Naam UC-01 Registering an account
Description A user creates a user-account
Actor Regular user
Precondition N/A
Scenario
  1. User clicks on the "Don't have an account yet? Click here!" link
  2. User fills in a valid Username and Password and clicks the "Create account" button
  3. The application validates whether an account with the credentials or username already exists.
  4. The application validates whether the input values match the minimum requirements.
  5. Upon acknowledgement of a valid registration attempt, the account will be created. The user will be taken to the Login page.
Postcondition The actor has a user-account and can log in.
Name UC-02 Logging in
Description An existing user logs in.
Actor Regular user
Precondition User has a user-account
Scenario
  1. User fills in the Username and Password fields correctly and clicks on the "Log in" button.
  2. The application validates the login attempt.
  3. Upon acknowledgement of a valid login attempt, the user will be logged in and greeted with the thread index page.
Postcondition The actor is logged in and loaded into the thread index page
